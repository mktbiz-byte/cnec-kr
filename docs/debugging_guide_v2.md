# cnecbiz.com 디버깅 가이드

## 1. 개요

이 문서는 cnecbiz.com 기업 홈페이지와 통합 관리자 시스템의 디버깅 가이드를 제공합니다. 개발자와 QA 팀이 시스템에서 발생할 수 있는 문제를 식별하고 해결하는 데 도움이 됩니다.

## 2. 디버깅 환경 설정

### 2.1 개발 환경 설정

```bash
# 개발 환경 설정
cd /home/ubuntu/cnec-kr
npm install
npm run dev
```

### 2.2 디버깅 도구

- **브라우저 개발자 도구**: Chrome DevTools, Firefox Developer Tools
- **React Developer Tools**: React 컴포넌트 디버깅
- **Redux DevTools**: 상태 관리 디버깅 (Redux 사용 시)
- **Supabase 대시보드**: 데이터베이스 및 인증 디버깅
- **Netlify 대시보드**: 배포 및 서버리스 함수 디버깅

## 3. 일반적인 문제 및 해결 방법

### 3.1 인증 관련 문제

#### 3.1.1 로그인 실패

**증상**: 사용자가 로그인할 수 없음

**디버깅 단계**:
1. 브라우저 콘솔에서 오류 메시지 확인
2. 네트워크 탭에서 로그인 요청 및 응답 확인
3. Supabase 대시보드에서 인증 로그 확인
4. 사용자 계정 상태 확인 (활성화 여부)

**일반적인 해결 방법**:
- 사용자 이메일 및 비밀번호 확인
- 계정 활성화 상태 확인
- Supabase 인증 설정 확인
- 토큰 만료 여부 확인

#### 3.1.2 권한 문제

**증상**: 사용자가 접근 권한이 있는 페이지에 접근할 수 없음

**디버깅 단계**:
1. 사용자 역할 및 권한 확인
2. 라우트 보호 로직 확인
3. 토큰 유효성 확인

**일반적인 해결 방법**:
- 사용자 역할 업데이트
- 권한 검사 로직 수정
- 토큰 갱신

### 3.2 데이터베이스 관련 문제

#### 3.2.1 데이터 조회 실패

**증상**: 데이터가 로드되지 않거나 빈 결과 반환

**디버깅 단계**:
1. Supabase 쿼리 로그 확인
2. RLS 정책 확인
3. 쿼리 구문 확인
4. 데이터 존재 여부 확인

**일반적인 해결 방법**:
- RLS 정책 수정
- 쿼리 구문 수정
- 인덱스 추가

#### 3.2.2 데이터 업데이트 실패

**증상**: 데이터 업데이트가 적용되지 않음

**디버깅 단계**:
1. Supabase 쿼리 로그 확인
2. RLS 정책 확인
3. 업데이트 구문 확인
4. 트리거 및 제약 조건 확인

**일반적인 해결 방법**:
- RLS 정책 수정
- 업데이트 구문 수정
- 제약 조건 확인

### 3.3 UI 관련 문제

#### 3.3.1 렌더링 문제

**증상**: 컴포넌트가 렌더링되지 않거나 잘못 렌더링됨

**디버깅 단계**:
1. React Developer Tools로 컴포넌트 트리 확인
2. props 및 state 확인
3. 조건부 렌더링 로직 확인
4. 콘솔 오류 확인

**일반적인 해결 방법**:
- 조건부 렌더링 로직 수정
- props 및 state 초기화 확인
- 키(key) 속성 확인

#### 3.3.2 스타일 문제

**증상**: 스타일이 적용되지 않거나 잘못 적용됨

**디버깅 단계**:
1. 브라우저 개발자 도구로 요소 검사
2. CSS 클래스 및 스타일 확인
3. 미디어 쿼리 확인
4. CSS 우선순위 확인

**일반적인 해결 방법**:
- CSS 클래스 수정
- !important 사용 (최후의 수단)
- 스타일 우선순위 조정

## 6. 문제 해결 가이드

### 6.1 일반적인 문제 해결 단계

1. 문제 식별: 정확한 문제 상황 파악
2. 정보 수집: 오류 메시지, 로그, 재현 단계 등 수집
3. 가설 수립: 문제의 원인에 대한 가설 수립
4. 가설 검증: 디버깅 도구 및 코드 검사를 통한 가설 검증
5. 해결책 적용: 문제 해결을 위한 코드 수정
6. 검증: 문제가 해결되었는지 확인
7. 문서화: 문제 및 해결 방법 문서화

### 6.2 자주 발생하는 문제 및 해결 방법

#### 6.2.1 "Cannot read property 'X' of undefined" 오류

**원인**: 객체가 undefined인 상태에서 속성에 접근하려고 함

**해결 방법**:
- 옵셔널 체이닝 사용: `obj?.prop`
- 기본값 설정: `obj || {}`
- 조건부 렌더링: `{obj && <Component prop={obj.prop} />}`

#### 6.2.2 무한 렌더링 루프

**원인**: 렌더링 중에 상태 업데이트가 발생하여 다시 렌더링되는 루프

**해결 방법**:
- useEffect 의존성 배열 확인
- 조건부 상태 업데이트 추가
- 메모이제이션 사용 (useMemo, useCallback)

#### 6.2.3 CORS 오류

**원인**: 다른 도메인의 리소스에 접근할 때 CORS 정책 위반

**해결 방법**:
- 서버에서 적절한 CORS 헤더 설정
- 프록시 서버 사용
- Netlify 함수 사용

#### 6.2.4 인증 토큰 만료

**원인**: 인증 토큰이 만료되어 API 요청 실패

**해결 방법**:
- 토큰 갱신 로직 구현
- 토큰 만료 시 자동 로그아웃 및 로그인 페이지로 리디렉션
- 토큰 만료 시간 모니터링
