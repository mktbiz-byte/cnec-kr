# CNEC 한국 사업자등록정보 인증 시스템 개선 프로젝트 완료 보고서

## 1. 프로젝트 개요

### 1.1. 프로젝트 목표 및 배경

본 프로젝트는 기존 CNEC 한국 사업자등록정보 인증 시스템의 사용자 경험을 개선하고, 관리 효율성을 높이는 것을 목표로 진행되었습니다. 주요 개선 요구사항은 다음과 같습니다.

- 사업자등록번호 기반의 간편한 로그인 및 회원가입 기능 구현
- 한국 사용자에게 친숙한 디자인 및 UI/UX 적용
- 관리자와 기업 관리자를 위한 전용 페이지 개발

### 1.2. 주요 개선 사항 요약

- **인증 시스템:** 사업자등록번호와 비밀번호만으로 로그인할 수 있도록 인증 절차를 간소화했습니다.
- **디자인:** 한국의 전통적인 색상과 모던한 디자인 요소를 결합하여 사용자 친화적인 UI를 구현했습니다.
- **관리 기능:** 관리자가 기업 회원을 효율적으로 관리할 수 있도록 승인, 조회, 상태 변경 등의 기능을 제공하는 관리자 페이지를 개발했습니다.
- **기업 기능:** 기업 회원이 자신의 정보를 확인하고 수정할 수 있는 기업 관리자 페이지를 개발했습니다.

## 2. 주요 기능

### 2.1. 사업자등록번호 기반 로그인 및 회원가입

- **회원가입:** 사업자등록번호 진위 확인 후, 이메일과 비밀번호 등 최소한의 정보만으로 회원가입이 가능합니다.
- **로그인:** 사업자등록번호와 비밀번호를 사용하여 간편하게 로그인할 수 있습니다.
- **역할 분리:** 관리자와 기업 관리자 로그인 탭을 분리하여 사용성을 높였습니다.

### 2.2. 관리자 페이지

- **대시보드:** 전체 기업 수, 승인 완료, 승인 대기 현황을 한눈에 파악할 수 있는 통계 정보를 제공합니다.
- **기업 회원 승인:** 신규 가입을 신청한 기업의 정보를 확인하고, 승인 또는 거절할 수 있습니다.
- **기업 관리:** 승인된 기업 목록을 조회하고, 검색 및 상태(활성/정지) 변경이 가능합니다.
- **관리자 계정 관리:** 등록된 관리자 계정 목록을 조회하고 관리할 수 있습니다.

### 2.3. 기업 관리자 페이지

- **대시보드:** 로그인한 기업의 상세 정보를 요약하여 보여줍니다.
- **프로필 관리:** 회사명, 대표자명, 연락처, 주소 등 기업 정보를 직접 수정할 수 있습니다.

### 2.4. 한국식 디자인 및 UI/UX

- **디자인 테마:** 청자색, 단청색 등 한국적인 색상 팔레트와 Noto Sans KR 폰트를 적용하여 시각적 편안함을 제공합니다.
- **반응형 디자인:** 데스크톱, 태블릿, 모바일 등 다양한 디바이스에서 최적화된 화면을 제공합니다.
- **직관적인 UI:** 사용자가 기능을 쉽게 이해하고 사용할 수 있도록 직관적인 레이아웃과 컴포넌트를 사용했습니다.

## 3. 기술 스택

| 구분 | 기술 | 비고 |
|---|---|---|
| 프론트엔드 | React, Vite, CSS | 사용자 인터페이스 구축 |
| 백엔드 (BaaS) | Supabase | 데이터베이스, 인증, 서버리스 함수 |
| 데이터베이스 | PostgreSQL | Supabase 내장 데이터베이스 |
| 배포 | Netlify | 프론트엔드 및 서버리스 함수 배포 |

## 4. 배포 가이드

### 4.1. Supabase 설정

1. Supabase 프로젝트 생성 후, `Database` > `SQL Editor` 메뉴로 이동합니다.
2. `cnec-kr/supabase/schema.sql` 파일의 내용을 복사하여 실행합니다.
3. `Authentication` > `Providers` 메뉴에서 `Email` 제공자를 활성화합니다.

### 4.2. Netlify 배포

1. GitHub에 프로젝트 코드를 푸시합니다.
2. Netlify에서 `New site from Git`을 선택하고, 해당 저장소를 연결합니다.
3. 빌드 설정을 다음과 같이 입력합니다.
   - **Build command:** `npm run build`
   - **Publish directory:** `dist`
4. 환경 변수를 설정합니다.
   - `VITE_SUPABASE_URL`: Supabase 프로젝트 URL
   - `VITE_SUPABASE_ANON_KEY`: Supabase Anon Key
   - `BUSINESS_API_KEY`: 사업자등록번호 API 키
5. `Deploy site` 버튼을 클릭하여 배포를 시작합니다.
