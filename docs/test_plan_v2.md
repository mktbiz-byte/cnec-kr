# cnecbiz.com 테스트 계획

## 1. 개요

이 문서는 cnecbiz.com 기업 홈페이지와 통합 관리자 시스템의 테스트 계획을 설명합니다. 테스트는 기능 테스트, 통합 테스트, 사용자 인터페이스 테스트, 성능 테스트, 보안 테스트로 구성됩니다.

## 2. 테스트 환경

- **개발 환경**: 로컬 개발 환경
- **스테이징 환경**: Netlify 스테이징 서버
- **프로덕션 환경**: Netlify 프로덕션 서버
- **데이터베이스**: Supabase
- **브라우저**: Chrome, Firefox, Safari, Edge

### 6.3 CI/CD 파이프라인

GitHub Actions를 사용하여 CI/CD 파이프라인을 구성합니다.

```yaml
name: CI/CD Pipeline

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main, develop ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Setup Node.js
        uses: actions/setup-node@v2
        with:
          node-version: '16'
      - name: Install dependencies
        run: npm ci
      - name: Run linting
        run: npm run lint
      - name: Run unit tests
        run: npm run test:unit
      - name: Run integration tests
        run: npm run test:integration
      - name: Build
        run: npm run build

  deploy:
    needs: test
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Setup Node.js
        uses: actions/setup-node@v2
        with:
          node-version: '16'
      - name: Install dependencies
        run: npm ci
      - name: Build
        run: npm run build
      - name: Deploy to Netlify
        uses: netlify/actions/cli@master
        with:
          args: deploy --prod
        env:
          NETLIFY_AUTH_TOKEN: ${{ secrets.NETLIFY_AUTH_TOKEN }}
          NETLIFY_SITE_ID: ${{ secrets.NETLIFY_SITE_ID }}
```
