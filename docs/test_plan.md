# 테스트 계획 및 체크리스트

## 개요

이 문서는 cnec.jp 웹사이트의 인증 시스템 개선, 캠페인 관리 및 결제 시스템 구축에 대한 테스트 계획과 체크리스트를 제공합니다. 각 기능별로 테스트 항목을 정의하고, 테스트 결과를 기록할 수 있도록 구성되어 있습니다.

## 1. 인증 시스템 테스트

### 1.1 관리자 로그인 시스템

| 테스트 항목 | 예상 결과 | 테스트 결과 | 비고 |
|------------|----------|------------|------|
| 관리자 로그인 페이지 접근 | `/auth/admin-login` 페이지가 정상적으로 로드됨 | | |
| 유효한 관리자 계정으로 로그인 | 로그인 성공 후 관리자 대시보드로 리디렉션 | | |
| 유효하지 않은 관리자 계정으로 로그인 | 로그인 실패 메시지 표시 | | |
| 관리자 권한 확인 | 관리자만 접근 가능한 페이지에 일반 사용자 접근 시 차단 | | |
| 로그아웃 기능 | 로그아웃 후 세션 종료 및 로그인 페이지로 리디렉션 | | |

## 테스트 실행 방법

1. 개발 환경에서 테스트:
   ```bash
   cd /home/ubuntu/cnec-kr
   npm run test
   ```

2. 스테이징 환경에서 테스트:
   - 스테이징 URL: https://staging.cnec.jp
   - 테스트 계정:
     - 관리자: admin@cnec.jp / password
     - 기업: company@example.com / password

3. 테스트 결과 기록:
   - 이 문서의 "테스트 결과" 열에 결과 기록
   - 문제 발견 시 GitHub 이슈 생성
